# 销售手册工具箱 — 案例公司设定文档

> 版本：v1.0
> 用途：作为手册所有模块示范范本的统一贯穿案例，供培训现场讲解、模板填写示例、场景演练脚本调用。
> 设计原则：两个案例代表B2B业务模式的两极（标准化/短周期/比价 vs 定制化/长周期/拼专业度），使高敏感模块（决策链、报价策略、跟进节奏、投标应对）的差异一眼可见；低敏感模块（企业介绍话术、产品BEDS逻辑、商务礼仪、售后服务标准）任选其一贯穿即可，无需重复演示。

---

## 案例公司甲——标准化产品 × 中小企业客户

| 维度 | 设定 |
|---|---|
| 行业/产品 | 工业耗材/标准化设备配件制造商（如工业密封件、标准紧固件类产品），SKU相对固定，可现货或短周期供货 |
| 目标客户 | 中小型制造企业（营收3000万-2亿），采购这类产品用于自身生产线 |
| 决策链 | 简单，1-2人：采购经理（实际决策）+ 老板（大额时过问），无复杂审批流程 |
| 决策周期 | 短，2-4周 |
| 采购逻辑 | 价格敏感、比价明显、看重交期稳定性和批量折扣 |
| 竞争格局 | 同类供应商多，同质化竞争，价格战常态 |
| 典型客户画像 | "XX精密制造"——年采购额200万，3家供应商比价，采购经理年度考核压成本 |

**适用模块提示**：报价谈判、异议处理（价格类）、跟进节奏（短周期）、客户分层（中小企业维度）

## 案例公司乙——定制化项目型产品 × 大企业/政府客户

| 维度 | 设定 |
|---|---|
| 行业/产品 | 工业自动化解决方案提供商（如产线自动化改造项目），非标定制，需现场勘测+方案设计 |
| 目标客户 | 大型制造企业集团或地方政府产业园区项目 |
| 决策链 | 复杂，4-6人：技术把关人（评估方案可行性）+ 采购/招标办（走流程）+ 财务（预算审批）+ 分管副总或分管领导（最终拍板）+ 有时还有第三方评审专家 |
| 决策周期 | 长，3-9个月，常涉及招投标流程 |
| 采购逻辑 | 不单纯比价，看重方案匹配度、过往案例背书、资质合规性、长期服务能力 |
| 竞争格局 | 供应商少而集中，拼方案专业度和案例背书，价格战空间小 |
| 典型客户画像 | "XX产业园管委会"或"XX集团华东分厂"——单笔项目300-800万，走公开招标或邀标流程，评审周期长 |

**适用模块提示**：决策链画像与分角色话术、投标应对、长周期跟进节奏、方案设计标准

## 调用规则

| 模块敏感度 | 调用方式 |
|---|---|
| 高敏感模块（决策链、客户分层标准、报价策略、跟进节奏、投标应对） | 甲、乙并列演示，展示同一工具模板在两种业务模式下的差异化填法 |
| 低敏感模块（企业介绍话术、产品BEDS逻辑、商务礼仪、售后服务标准流程） | 任选甲或乙其一贯穿即可，不重复演示 |

## 说明

两个案例公司均为虚构设定，不涉及任何真实企业名称或品牌，仅用于培训场景中的示范填写，参训企业实际共创产出时需替换为自身真实业务信息。

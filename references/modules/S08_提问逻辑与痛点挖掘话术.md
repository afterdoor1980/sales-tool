# S08 提问逻辑与痛点挖掘话术

> 版本：v1.0（正式稿）
> 篇章：六、需求挖掘篇
> 模板类型：类型A（话术库）
> 适用业务模式：通用（提问框架一致，深挖内容按甲乙调整）
> 前置依赖：S05决策链画像（对谁问、问什么优先级不同）、S03（信息不足状态的处理逻辑）
> 模块目的：让销售能挖出客户真实需求和顾虑，而非表面需求，为后续方案和报价打好基础

---

## 一、本阶段沟通前准备要点（对应S07的模块范围承诺）

S07（拜访前准备清单）与本模块的关系是"准备"与"执行"之别，而非两次分开会面之别——**很多情况下，S08的需求挖掘提问就发生在S07准备的同一次拜访里**，尤其甲类客户常见的情况是简短介绍后直接进入需求挖掘。因此S07的拜访前准备工作，本身就应当包含对本节内容的预习，而不是假设需求挖掘一定要等到"下一次"拜访。乙类客户由于决策链复杂，确实更可能把初步接触和深入需求挖掘分成不同会面，但这不是强制规则，需结合实际情况判断。

本节内容：

S07（拜访前准备清单）明确说明"需求挖掘/方案讲解会前的准备见S08"，本节兑现这一承诺：

| 准备事项 | 说明 |
|---|---|
| 提前明确本次要挖掘的重点方向 | 结合S05决策链地图，确认本次沟通对象的角色关注点，提前想好该从哪个提问阶段切入（不必每次都从背景类问起，若已有初步信息可直接从难点类开始） |
| 准备好追问路径的心理预期 | 提前预想客户可能给出的2-3种回答方向，分别对应哪种追问路径，避免临场反应不过来 |
| 明确本次沟通的角色边界 | 若本次沟通对象只是决策链中的一环（如仅技术把关人在场），提前想清楚哪些信息需要后续桥接给其他角色，而非本次一次性问全 |

## 二、提问逻辑总览（含追问路径）

| 提问阶段 | 目的 | 首层提问后的追问路径 |
|---|---|---|
| 背景类 | 了解现状 | 若客户回答简略→追问具体细节（数量/频率/流程）；若客户主动多说→顺势引导到难点类提问 |
| 难点类 | 挖掘具体问题 | 若客户说出一个难点→追问"这个情况多久出现一次""具体是哪个环节"；若客户说"还好、没什么大问题"→转换角度问对比性问题 |
| 影响类 | 让客户意识到难点的实际代价 | 若客户轻描淡写→追问量化影响；若客户已经表现出重视→可直接推进到方向确认 |
| 方向确认类 | 确认解决方向 | 若客户认同方向→可自然过渡到方案设计环节；若客户犹豫或修正方向→回到难点类重新挖掘 |

**通用兜底规则**：若同一阶段追问2-3轮后客户依然给不出实质内容（并非防御性回避，而是对方确实缺乏相关信息，如基层对接人不掌握全局情况），不再继续追问，按S03/S05已建立的"信息不足"逻辑处理——记录为信息不足，更新至S05决策链地图，转向其他更合适的角色获取该信息，或留待后续接触补充，不在当次对话中反复逼问。

## 三、案例公司甲（采购经理）

**S08-01｜背景类**

| 字段 | 内容 |
|---|---|
| 编号 | S08-01 |
| 首层提问 | "目前您这边这个品类大概是怎么采购的？是固定几家供应商还是会根据情况调整？" |
| 追问路径-简略回答 | "大概多久会有一次调整供应商的情况？主要是什么原因促使您考虑更换？" |
| 追问路径-详细回答 | 顺势追问难点类（S08-02）："这几家供应商里，有没有哪家在某些方面让您觉得可以再改善？" |
| 禁忌话术 | 不要一开始就问"您预算多少"，过早问预算会显得唐突且引发戒备 |

**S08-02｜难点类**

| 字段 | 内容 |
|---|---|
| 编号 | S08-02 |
| 首层提问 | "现在合作的供应商，有没有什么地方您觉得不太满意，或者希望能改善的？" |
| 追问路径-说出难点 | "这个情况大概多久发生一次？是所有订单都这样还是特定情况下才会？" |
| 追问路径-回避/轻描淡写 | "那如果单纯从效率角度，您觉得整个采购流程里，有没有哪个环节其实可以更顺畅一些？" |
| 禁忌话术 | 不要直接问"你们现在的供应商是不是不行"，带有引导性和贬低竞品的暗示 |

**S08-03｜影响类**

| 字段 | 内容 |
|---|---|
| 编号 | S08-03 |
| 首层提问 | "如果交期不稳定的情况持续，对您这边的生产排期影响大吗？" |
| 追问路径-轻描淡写 | "大概会导致停线多久？这种情况一年大概出现几次？" |
| 追问路径-表现出重视 | 可直接推进方向确认类提问（S08-04），不需要继续深挖影响 |
| 禁忌话术 | 不要用"这样不是很麻烦吗"这种带评判色彩的反问，容易让客户觉得被说教 |

**S08-04｜方向确认类**

| 字段 | 内容 |
|---|---|
| 编号 | S08-04 |
| 首层提问 | "如果我们能在交期稳定性上给您更明确的保障，是不是会优先考虑我们？" |
| 追问路径-认同 | 自然过渡到方案/报价环节 |
| 追问路径-犹豫或提出其他条件 | "除了交期，还有没有其他您比较看重的地方？"（回到难点类S08-02重新挖掘被忽略的维度） |
| 禁忌话术 | 不要在客户还没充分表达难点时就急于问这个，显得准备不足 |

## 四、案例公司乙（技术把关人→分管领导，含角色跳转说明）

**角色跳转说明（重要前提）**：技术把关人和分管领导通常**不在同一场对话中**，销售需要在两者之间做信息桥接，而非假设分管领导凭空知道技术层面已经聊过什么。具体操作：完成S08-05至S08-07与技术把关人的对话后，销售应先将了解到的产线情况和技术把关人的态度**整理成简要书面/口头汇报材料**，在安排与分管领导的沟通前先行同步（可结合S05"关键角色接触受限时的应对策略"一并使用），确保S08-08开场时"基于目前了解到的产线情况"这句话有真实信息支撑，而非无中生有。

**S08-05｜背景类（技术把关人）**

| 字段 | 内容 |
|---|---|
| 编号 | S08-05 |
| 首层提问 | "咱们这条产线目前的自动化程度大概是怎样的？现在主要还是靠人工哪些环节？" |
| 追问路径-简略回答 | "这些人工环节大概占整体产能的多大比例？" |
| 追问路径-详细回答 | 顺势追问难点类（S08-06）："这些环节里，有没有因为人工操作导致过一致性或效率上的问题？" |
| 禁忌话术 | 不要用过于笼统的"你们现在技术怎么样"，显得不专业、不了解对方产线情况 |

**S08-06｜难点类（技术把关人）**

| 字段 | 内容 |
|---|---|
| 编号 | S08-06 |
| 首层提问 | "有没有遇到过因为设备/工艺问题导致停产或者效率打折扣的情况？大概多久发生一次？" |
| 追问路径-说出难点 | "这个问题具体是哪个环节触发的？之前有没有尝试过什么解决方式，效果怎么样？" |
| 追问路径-回避 | "那如果从整体效率角度，您觉得目前产线有没有还能再优化的空间？" |
| 禁忌话术 | 不要问过于笼统的"有什么问题吗"，技术把关人需要具体、专业的问题才愿意展开 |

**S08-07｜影响类（技术把关人）**

| 字段 | 内容 |
|---|---|
| 编号 | S08-07 |
| 首层提问 | "这种停产/效率问题，大概会造成多大的产能损失？对整体交付计划有没有影响？" |
| 追问路径-轻描淡写 | "如果按年度计算，这种情况大概会造成多少产能上的损失？" |
| 追问路径-表现出重视 | 可推进到方向确认，或建议邀请分管领导一同讨论（进入S08-08前先完成"角色跳转说明"中的信息桥接动作） |
| 禁忌话术 | 不要替对方下结论说"这肯定影响很大"，让客户自己说出实际影响更有说服力 |

**S08-08｜方向确认类（分管领导）**

| 字段 | 内容 |
|---|---|
| 编号 | S08-08 |
| 前提动作 | 开场前需已完成与技术把关人信息的桥接同步（见本节"角色跳转说明"），确保本轮提问的信息基础真实存在 |
| 首层提问 | "基于目前了解到的产线情况，如果我们能提供一个既解决效率问题、又能控制在预算范围内的方案，是不是符合您这边的整体考虑？" |
| 追问路径-认同 | 自然过渡到方案设计环节，可提议安排现场勘测 |
| 追问路径-犹豫或修正方向 | "除了效率提升，在预算或实施周期上，您这边有没有特别的顾虑？"（回到难点类重新挖掘） |
| 禁忌话术 | 不要在信息了解不充分时就急于问方向确认类问题，容易显得准备不足、想赶紧成交 |

## 五、客户防御性强/不愿深入回答时的应对

| 情况 | 应对方式 |
|---|---|
| 客户简单带过、不愿展开（如只说"还行吧"） | 换一个更具体的场景化提问（如"比如说上个月有没有类似情况"），具体场景比抽象问题更容易打开话匣子 |
| 客户明显有所保留（涉及内部信息或竞品比较） | 不强行追问，转而通过后续多次接触逐步建立信任后再了解，参照S05内部教练识别方法 |
| 追问2-3轮仍无实质内容（非防御，而是确实不掌握信息） | 参照本文档第二节通用兜底规则，记录为信息不足，转向其他角色或后续接触 |

## 六、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| 提问逻辑顺序正确 | 是否按"背景→难点→影响→方向确认"的顺序设计，不能一上来就是封闭式提问 |
| 追问路径完整 | 每个首层提问是否配有至少两种典型回答对应的追问路径，而非只有一层问题 |
| 问题具体不笼统 | 是否针对具体场景提问，而非"有什么需求吗"这种笼统问法 |
| 角色针对性体现 | 面对不同角色的提问内容是否体现其关注点差异（参照S05） |
| 角色跳转是否交代清楚 | 涉及多角色接力提问的场景，是否说明信息如何在角色间桥接，而非默认后一角色凭空知晓前情 |
| 兜底规则是否存在 | 是否说明追问多轮仍问不出实质内容时该怎么办，而非假设每次追问都必然有效 |
| 编号完整 | 每条提问条目是否有唯一编号，格式是否符合字段规范 |

## 七、课件页要点

1. 讲解"先开放后封闭"的提问逻辑，避免学员把提问做成审讯式的一连串封闭式问题。
2. 强调追问路径不是死板脚本，而是"根据对方回答走不同分支"的决策树思维，需要现场判断客户属于哪种回答类型。
3. 讲解乙场景下的角色跳转和信息桥接，避免学员想当然地认为可以对分管领导直接说"基于我们聊的情况"却没有做过信息同步。
4. 讲解信息不足时的兜底规则，避免学员对同一个人反复逼问同一个问题，造成客户不适。
5. 现场练习：让学员两两一组，一人扮演客户、一人按提问逻辑现场挖掘"痛点"，导师点评提问顺序、追问路径选择是否恰当、遇到"问不出"时是否正确转入兜底处理。

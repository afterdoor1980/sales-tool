# S01 企业介绍标准话术

> 版本：v1.0（正式稿）
> 篇章：一、信任背书篇
> 模板类型：类型A（话术库）
> 适用业务模式：通用（以案例公司甲为示范案例，乙可参照替换背景信息）
> 适用客户分层：不区分（企业介绍是面向任何对象的基础动作，分层影响的是后续深入程度而非介绍本身）
> 前置依赖：字段规范文档 v1.5、案例公司设定文档
> 模块目的：让任何一个销售（尤其新人）在不同场合都能一致、专业地介绍公司，不因人而异导致客户对企业认知混乱

## 一、企业如何将示范内容替换为自己真实信息（通用操作提示，后续同类模块可参照）

本模块及后续所有以案例公司甲/乙为示范的模块，参训企业在实际使用时需按以下步骤替换为自身真实内容，而非照抄案例原文：

| 步骤 | 操作 |
|---|---|
| 1. 定位可替换要素 | 找出话术中的具体信息点（如成立年限、产能数据、合作客户案例、核心优势描述） |
| 2. 逐项替换为真实数据 | 用本企业真实的成立时间、产能、客户案例（如无法透露客户名称，可用"某XX行业头部企业"代替）逐一替换 |
| 3. 保留话术的逻辑结构 | 替换的是具体信息，不是话术的组织逻辑（如"公司历程→核心实力→差异化优势→合作意向"这个顺序应当保留），逻辑结构是经过设计的，不建议自行打乱 |
| 4. 验收时对照检查 | 参照本模块验收细则中的"案例贴合度"，检查是否只是换了名字的照抄，还是真正体现了本企业的实际情况 |

## 二、被打断后如何接回（三版本通用，实战高频状况）

企业介绍是主动输出场景，现实中经常被打断（如对方接电话、展会上被岔开话题、会议中途有人插话），需要有自然的接回方式，而非从头重讲或尴尬冷场。

| 情况 | 应对方式 |
|---|---|
| 对方因外部原因中断（接电话、临时被叫走） | 等对方回来后，用一句话概括刚才讲到哪里再继续，如"刚才说到我们在交期稳定性上的优势，我接着说"，不要从头重讲一遍 |
| 对方主动打断并提问（对某个点感兴趣，提前追问） | 顺势回答该问题，回答完后用一句话说明"刚才是想大概介绍一下……"再简要带过原本还没讲完的部分，不必强行讲完剩余内容 |
| 对方打断后话题被带偏 | 不强行拉回话题，先跟随对方的话题走1-2轮，再自然寻找机会衔接回来（如"说到这个，其实也是我们XX优势的一部分"），避免让对方觉得你只顾自己讲 |

## 三、正文

### S01-01｜30秒版

| 字段 | 内容 |
|---|---|
| 场景/触发条件 | 展会偶遇、电话初次接触、社交场合简短问候等碎片化场景，对方尚未表现出明确兴趣 |
| 对话对象角色 | 通用（任何初次接触对象） |
| 标准话术 | "我们是做工业密封件和标准紧固件的，成立12年了，主要服务中小型制造企业，像XX、XX这些企业都是我们长期合作的客户，交期稳定是我们最大的优势。" |
| 禁忌话术 | 不要一上来堆砌资质证书或荣誉称号（对方没兴趣听细节）；不要说"我们是行业领先"这种空泛自夸而不给具体依据；**不要在30秒内塞入超过3个信息点**（如同时讲产能+资质+案例+优势），信息过载会导致对方一个都记不住，反而失去这个版本"简短有记忆点"的意义；**不要语速过快像背台词**，30秒版本尤其容易因为想在短时间内说完而语速失控，显得不自然、不真诚 |
| 话术意图（四维度） | **情绪**：让对方在极短时间内产生"值得多聊两句"的兴趣，而非戒备或不耐烦；**信息目标**：传递"做什么、服务谁、有什么可信依据"三个最核心信息；**关系**：不过度推销，保持轻松、专业的第一印象；**行动导向**：让对方主动问下一句（如"具体是做什么产品的"），而非听完就结束对话 |
| 升级路径 | 若对方追问细节或表现出兴趣（如问"具体做哪些产品"）：自然过渡到3分钟版本；若对方只是礼貌性回应无进一步兴趣：交换联系方式即可，不强行推进 |
| 使用注意 | 语速适中，突出1-2个最有记忆点的信息（如"12年""交期稳定"），不要试图塞入过多信息 |

### S01-02｜3分钟版

| 字段 | 内容 |
|---|---|
| 场景/触发条件 | 客户主动约见、初次拜访开场、电话沟通中客户明确表示"想多了解一下" |
| 对话对象角色 | 采购经理或对接窗口 |
| 标准话术 | "我们公司成立12年，专注工业密封件和标准紧固件的研发生产，目前有两条自动化产线，年产能可以覆盖大部分中小型制造企业的常规需求。我们和XX、XX这些企业合作都在5年以上，核心优势是72小时紧急补货能力和稳定的批次质检记录。今天想先了解一下您这边具体的采购需求和目前遇到的痛点，看看我们能不能提供合适的方案。" |
| 禁忌话术 | 不要变成单向输出、讲完就结束，3分钟版本的关键是讲完后自然过渡到需求了解，不能只讲不问；不要在这个阶段就开始报价；**不要罗列超过3个客户案例**，1-2个有代表性的即可，堆砌案例名称会显得像在炫耀而非举证；**不要用"我们规模很大""我们很专业"这类无数据支撑的形容词**，凡是形容词都应尽量替换为具体数字或事实（如用"两条自动化产线"代替"生产能力强"） |
| 话术意图（四维度） | **情绪**：建立初步专业信任，让客户感到"这是一家靠谱、有底子的供应商"；**信息目标**：传递产能、稳定性、合作案例这些实质性信息，同时获取客户当前采购状况的初步信息；**关系**：从"自我介绍"自然过渡到"了解对方"，避免单向推销的观感；**行动导向**：引导客户开口说出自己的需求或痛点，进入需求挖掘环节 |
| 升级路径 | 若客户是决策层（如老板亲自接待）或表现出深入合作意向：可过渡到10分钟版本，加入更完整的公司实力展示；若客户明确只是初步了解、无深入意向：3分钟版本讲完即可，转入常规跟进节奏 |
| 使用注意 | 结尾必须落在"了解客户需求"这个动作上，不能只是自我介绍完就等对方说话 |

### S01-03｜10分钟版

| 字段 | 内容 |
|---|---|
| 场景/触发条件 | 正式的公司介绍会、决策层深度沟通、方案汇报前的实力展示环节 |
| 对话对象角色 | 老板/总经理或其他决策层人员 |
| 标准话术 | 结构分四段：① 公司历程（成立12年，从XX起步到现在两条自动化产线的发展过程）② 核心实力（产能数据、质检体系、过往5年以上长期合作客户案例）③ 差异化优势（72小时紧急补货能力、批次质检报告可追溯、定制化包装/交期方案）④ 合作意向（"基于您这边的规模和需求，我们认为双方在XX方面有很好的合作空间，接下来想具体了解您的采购计划"） |
| 禁忌话术 | 不要把10分钟版本做成"3分钟版本的简单拉长"，必须包含决策层关心的内容（如长期合作稳定性、供应链风险应对能力），而非重复产品参数；不要通篇自夸，需穿插具体案例和数据支撑；**不要四段内容平均用力**，核心实力和差异化优势应占更大篇幅（决策层最关心"你能不能长期稳定地支撑我"），公司历程部分点到为止即可 |
| 话术意图（四维度） | **情绪**：让决策层感到"这不是一次性推销，而是在评估长期合作伙伴"；**信息目标**：传递完整的企业实力画像，同时获取决策层关注的深层顾虑（如供应链风险、长期价格稳定性）；**关系**：建立决策层级别的信任，为后续更大额度或框架合作打基础；**行动导向**：推进到具体的合作意向讨论或采购计划了解 |
| 升级路径 | 讲完后若决策层表现出进一步兴趣：直接进入需求挖掘或方案设计环节（对应后续模块，待模块产出后补充具体编号引用）；若发现决策层对某类顾虑（如供应链风险）反复关注：转入对应异议处理，可参照S11相关条目 |
| 使用注意 | 需提前了解对方企业的基本背景（行业、规模），讲的时候适当穿插能引起共鸣的案例（如同行业客户合作案例），而非通用模板一成不变 |

## 四、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| 三版本时长匹配 | 30秒版本是否真的能在30秒内说完（不是写了一段话但实际要读1分钟），3分钟、10分钟同理 |
| 三版本内容递进而非重复 | 10分钟版本不能只是3分钟版本的简单重复加长，应包含决策层关心的额外内容 |
| 升级路径自然 | 版本间的过渡是否自然（如30秒版本结尾是否留出让对方追问的空间），而非生硬转折 |
| 案例贴合度 | 提及的客户合作案例、数据是否基于本企业真实情况改写（参照本文档"企业如何将示范内容替换为自己真实信息"一节），而非直接照抄案例公司甲的示范原文 |
| 被打断应对是否掌握 | 现场演练时故意打断学员，检查其能否自然接回，而非从头重讲或冷场 |

---

## 五、课件页要点

1. 强调"三个版本不是可以随便选，而是要匹配场景"——展会上说10分钟版本会把人吓跑，决策层会议上只说30秒版本会显得不够专业。
2. 讲解"被打断后如何接回"，这是实战高频状况，很多销售因为一次被打断就彻底乱了节奏，需要提前演练。
3. 现场练习：让学员分别准备本企业的30秒/3分钟/10分钟版本，同桌互相扮演"客户"进行测试，其中至少安排一次"故意打断"环节，导师用验收细则点评时长、内容匹配度和被打断后的应对表现。

## 一、异议识别与拆解指南（使用本模块前必读）

客户的真实表述往往模糊、混合，或是委婉说法，不会自报"我这是价格异议"。在套用具体异议类型话术前，需先做识别与拆解，避免误判类型导致话术答非所问。

### 常见模糊/伪装表述对照表

| 客户原始表述 | 可能对应的真实异议类型 | 澄清提问 |
|---|---|---|
| "我再考虑一下" | 预算延迟 / 信任顾虑 / 单纯敷衍 | "方便问一下，是预算上需要再申请，还是对方案本身还有顾虑？" |
| "你们比XX贵" | 价格类 / 信任顾虑（怀疑贵得不合理） | "您是已经拿到XX的具体报价在对比，还是担心我们这个价格里有什么不透明的地方？" |
| "我们内部还要再讨论" | 决策延迟（甲乙均可能）/ 委婉拒绝 | "讨论主要是哪个环节，是技术评估还是预算审批？我这边可以提供针对性材料。" |
| "之前没合作过，不太了解你们" | 信任顾虑 / 单纯需要更多信息（非真正异议） | "您比较在意的是我们的资质案例，还是想先小范围试一下？" |
| "XX厂也在跟我们谈，条件比你们好" | 竞品对比类 / 借竞品压价（本质仍是价格类） | 视具体后续话语判断，通常需先确认对方条件是否书面化；**若客户反复强调"总之更便宜"，本质是价格类异议，应转回价格类话术处理** |

### 复合异议处理规则

当客户一句话同时包含多类信号时（如"你们价格贵，而且之前没听说过你们"——同时是价格+信任）：

1. **先处理占比更大/更根本的那类**（可参考情绪强度判断，如陌生信任感通常比价格更根本，信任建立不起来，价格谈了也没用）
2. **用一句话搭桥，再进入第二类的话术**（如先给信任类回应建立基础信任，再用过渡语引出价格类价值论证）
3. **不在一次回应里堆砌两类完整话术**，避免让客户感觉在被"话术攻势"应对，而非被认真倾听

## 二、案例公司甲（标准化产品/中小企业客户）

### 异议类型：价格类

**角色分支1：采购经理**

| 字段 | 内容 |
|---|---|
| 编号 | S11-01 |
| 场景/触发条件 | 客户说"你们价格比XX厂贵，能不能再便宜点" |
| 对话对象角色 | 采购经理 |
| 标准话术 | "理解您想控制成本。我们这个价格里包含了每批次的质检报告和72小时紧急补货承诺——如果按单价对比，XX厂能做到吗？我们可以看看您的采购量，是否能通过年度框架协议锁定更优价格。" |
| 禁忌话术 | 不要说"我们价格已经是最低了"；不要主动大幅让价 |
| 话术意图（四维度） | **情绪**：把"你在压价"的对峙感，转为"我们一起算总账"的合作感；**信息目标**：摸清客户真实比价对象和预算上限；**关系**：不破坏价格锚点，避免影响后续所有客户的议价基准；**行动导向**：引导客户接受"年度框架"这个替代方案，而非当场要求降价 |
| 升级路径 | 若客户仍要求直接降价：转向"年度框架协议"让价（而非单笔让价），让价幅度控制在预设区间内；若客户提出的采购量确实达到框架门槛，转交主管确认具体折扣比例——超出本人报价权限即转交，而非自行拍板 |
| 使用注意 | 甲客户决策快、比价直接，需在异议提出后立即回应，拖延会被解读为"心虚" |

**角色分支2：老板/总经理（大额订单时出现）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-02 |
| 场景/触发条件 | 大额订单超出采购经理审批权限，老板亲自过问价格 |
| 对话对象角色 | 老板/总经理 |
| 标准话术 | "这个价格是按您年采购量测算的最优区间。相比短期压价，我更想聊聊怎么帮您把整个供应链的稳定性和成本可预测性做起来——换供应商的隐性成本，往往比价格差更高。" |
| 禁忌话术 | 不要像对采购经理那样堆砌参数对比，老板更在意大局而非单价明细 |
| 话术意图（四维度） | **情绪**：从"被压价"转为"被当作战略伙伴对待"；**信息目标**：了解老板对供应链稳定性/换供应商风险的真实顾虑；**关系**：让老板感到被区别于普通采购对待，建立更高层级信任；**行动导向**：促成签订年度框架协议，而非停留在单笔议价 |
| 升级路径 | 若老板仍不满意战略合作说法、坚持要具体折扣数字：可提供"首年框架价"作为有条件让步（次年恢复标准价），需转交主管确认——原则上老板层级的让价决策不由一线销售单独拍板 |
| 使用注意 | 该分支只在大额订单、老板亲自介入时使用，不用于日常小额订单 |

> 说明：采购经理关心单价和参数对比，老板关心供应链战略价值，两者关注点实质不同，符合角色分支规则，予以拆分。

### 异议类型：预算/决策延迟类

| 字段 | 内容 |
|---|---|
| 编号 | S11-03 |
| 场景/触发条件 | 客户说"我再考虑一下，过阵子联系你" |
| 对话对象角色 | 采购经理。**说明**：此场景下老板通常不直接介入日常拖延决策（只有金额超出采购经理权限时才会介入，对应价格类的"老板版"），故本场景不做角色拆分——这一判断依据与乙的预算延迟类保持一致的说明逻辑 |
| 标准话术 | "没问题，方便问一下是预算还是交期上有顾虑吗？我们这批货源紧张，如果这周内确认能帮您锁定当前批次和价格。" |
| 禁忌话术 | 不要追问"到底什么时候能定"；不要立即报出更低价格试图逼单 |
| 话术意图（四维度） | **情绪**：把客户的"敷衍拖延"转化为"说出具体顾虑"；**信息目标**：判断拖延的真实原因是预算还是交期，而非笼统的"再看看"；**关系**：不施压，保持专业形象；**行动导向**：让客户在本周内给出明确答复，而非无限期拖延 |
| 升级路径 | 若客户超过约定的"本周内确认"仍未回复：进行第二轮跟进，改用"稀缺性提醒"而非重复价格论证（如告知库存/批次即将排满）；若连续两轮跟进无反馈，转入S14长周期跟进节奏表的常规跟进节奏，不再视为单独异议处理 |
| 使用注意 | 甲客户决策周期短（2-4周），此话术适合在报价后1周内跟进使用 |

### 异议类型：信任/风险顾虑类

| 字段 | 内容 |
|---|---|
| 编号 | S11-04 |
| 场景/触发条件 | 客户说"没听说过你们厂，之前一直用XX牌子，质量和交期能保证吗" |
| 对话对象角色 | 采购经理（此类异议通常由采购经理直接把关，不做角色拆分） |
| 标准话术 | "理解您的顾虑，换供应商确实有风险。这是我们最近半年给XX、XX两家同行业客户的供货记录和质检报告，您可以直接联系他们的采购负责人核实。另外我们可以先从小批量试单开始，验证稳定后再逐步加大订单量。" |
| 禁忌话术 | 不要只说"我们质量很稳定"却拿不出可验证材料；不要一上来就要求大批量下单 |
| 话术意图（四维度） | **情绪**：从"陌生供应商的不安全感"转为"有据可查、风险可控"的踏实感；**信息目标**：了解客户之前用老供应商时具体在意的指标是质量、交期还是价格稳定性；**关系**：用"小批量试单"降低客户决策门槛，建立循序渐进的信任路径；**行动导向**：促成客户同意小批量试单，而非要求立即大额签约 |
| 升级路径 | 若客户对小批量试单仍犹豫：可进一步提供"不满意可退货"条款作为兜底；若客户明确表示"需要看到更大客户案例"而非同行小客户，转交主管，由主管提供更大规模客户的背书材料 |
| 使用注意 | 需随身准备好同行业可核实的供货记录，试单方案要提前想好具体的量级和价格 |

### 异议类型：竞品对比类

| 字段 | 内容 |
|---|---|
| 编号 | S11-05 |
| 场景/触发条件 | "XX厂也在跟我们谈，他们说交期比你们快，还能免费送货上门" |
| 对话对象角色 | 采购经理（不做角色拆分） |
| 标准话术 | "交期和运费确实是可以比较的点，方便问一下他们承诺的交期是书面写进合同还是口头说的？我们的交期有违约赔偿条款兜底，这个XX厂能做到吗？免费送货我们也可以按您的年采购量谈进合同里。" |
| 禁忌话术 | 不要直接贬低竞品；不要因为对方口头承诺就跟着盲目承诺同等条件 |
| 话术意图（四维度） | **情绪**：从"客户拿竞品施压"的对峙感，转为"帮客户理性拆解对比条件"的专业感；**信息目标**：确认竞品承诺是否书面化、是否可持续；**关系**：不诋毁竞品，维护自身专业形象；**行动导向**：引导客户要求竞品也书面化承诺，同时提出自己可对等或更优的书面条款 |
| 升级路径 | 若客户仍坚持竞品条件优于我方且态度强硬：可申请针对该客户的"一次性竞对应对折扣"，但需转交主管审批，不由一线销售自行承诺 |
| 使用注意 | 需提前了解本地区常见竞品的实际交付能力。**需先判断客户核心诉求**：如果客户反复强调"总之XX厂更便宜"，说明本质是价格类异议被包装成竞品对比，应转回价格类话术处理，而非在交期/服务条款上纠缠 |

## 三、案例公司乙（定制化项目/大企业政府客户）

### 异议类型：价格类

**角色分支1：财务/预算审批方**

| 字段 | 内容 |
|---|---|
| 编号 | S11-06 |
| 场景/触发条件 | 方案评审会上，财务质疑"报价比其他投标方高" |
| 对话对象角色 | 财务/预算审批方 |
| 标准话术 | "我们的报价包含了现场勘测定制方案、验收后一年免费维护、以及本地化服务团队响应——建议对比全生命周期成本而非单纯采购价。附上的案例是我们在XX项目为客户节省的后期维护成本明细，供参考。" |
| 禁忌话术 | 不要单纯比较报价数字本身；不要在评审会现场承诺降价 |
| 话术意图（四维度） | **情绪**：从"你们比别人贵"的质疑，转为"帮财务算清楚账"的专业感；**信息目标**：了解财务审批的具体预算上限和考核指标；**关系**：不破坏投标公平性承诺，避免留下私下让价的口实；**行动导向**：让财务在评审报告中采纳"全生命周期成本"这一评估维度 |
| 升级路径 | 若财务坚持只比较采购价、不认可全生命周期成本说法：转交项目负责人（而非一线销售继续说服），由项目负责人与财务做进一步的书面成本核算说明会 |
| 使用注意 | 需提前准备好书面案例材料，财务角色更信书面数据而非口头说辞 |

**角色分支2：技术把关人**

| 字段 | 内容 |
|---|---|
| 编号 | S11-07 |
| 场景/触发条件 | 技术把关人质疑"报价高，是不是配置超出实际需求" |
| 对话对象角色 | 技术把关人 |
| 标准话术 | "这个配置是根据您现场勘测的实际产线情况定制的，不是标准套餐往上堆料。我可以带您逐项过一遍配置清单，哪些是必须项、哪些是可选项，您可以按实际需求调整。" |
| 禁忌话术 | 不要用"这是我们的标准方案"搪塞 |
| 话术意图（四维度） | **情绪**：从"被过度推销"的警惕，转为"方案是为我定制的"的信任；**信息目标**：了解技术把关人认为哪些配置是"不必要的"；**关系**：建立技术层面的专业可信度；**行动导向**：让技术把关人认可配置清单，在评审意见里给出技术支持性结论 |
| 升级路径 | 若技术把关人仍认为配置有冗余：安排现场复勘或第三方技术评估，不在无更多依据的情况下继续口头争论 |
| 使用注意 | 需随身携带可现场展示的配置清单和勘测记录 |

> 说明：财务关注总成本合理性，技术把关人关注配置必要性，两者关注点实质不同，符合角色分支规则，予以拆分。

### 异议类型：信任/风险顾虑类

**角色分支1：分管领导（关注问责与整体风险）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-08 |
| 场景/触发条件 | 分管领导问"你们之前有没有做过同类规模项目，出问题谁负责" |
| 对话对象角色 | 分管领导 |
| 标准话术 | "我们在XX集团做过类似规模的自动化改造项目，可以安排您和该项目的甲方对接人直接沟通。合同里会明确验收标准和质保条款，出现问题由我方在质保期内无偿处理。" |
| 禁忌话术 | 不要含糊回应"我们经验很丰富"却拿不出具体案例或联系人；不要口头承诺无限责任 |
| 话术意图（四维度） | **情绪**：从"决策风险焦虑"转为"责任边界清晰、有保障"的安心；**信息目标**：了解分管领导最担心的具体风险点；**关系**：建立可验证的第三方背书信任，而非依赖个人担保；**行动导向**：促成分管领导同意进入合同条款谈判阶段 |
| 升级路径 | 若分管领导对现有案例仍不满意（如认为规模/行业不够对标）：转交公司高层，由更高层级出面提供更强背书（如战略合作意向书），一线销售不适合承诺超出自己权限的责任条款 |
| 使用注意 | 必须配合真实可提供的案例清单和联系人 |

**角色分支2：技术把关人（关注技术可行性证明，非问责）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-09 |
| 场景/触发条件 | 技术把关人问"你们的方案在我们这种产线条件下真的可行吗，有没有验证过" |
| 对话对象角色 | 技术把关人 |
| 标准话术 | "我们在勘测阶段已经模拟过您这条产线的实际参数，可以提供仿真测试报告。同类工况的XX项目也做过验证，我可以安排您和对方的产线负责人直接交流细节。" |
| 禁忌话术 | 不要只强调"公司实力强"，需要具体的验证数据和同行交流机会 |
| 话术意图（四维度） | **情绪**：从"技术可行性存疑"转为"有数据支撑、可验证"的踏实感；**信息目标**：了解技术把关人具体质疑的是哪个技术环节；**关系**：建立技术层面的长期专业信任；**行动导向**：促成技术把关人在评审中给出技术可行性认可意见 |
| 升级路径 | 若技术把关人对仿真报告仍有疑虑：安排技术把关人与自家技术团队直接对话，转交技术负责人对接，不由销售继续解释技术细节 |
| 使用注意 | 需提前准备仿真报告等具体技术文件 |

> 说明：分管领导关心问责和整体风险，技术把关人关心技术验证，两者关注点实质不同，符合角色分支规则，予以拆分。

### 异议类型：竞品对比类

**角色分支1：招标办/采购流程负责人（关注合规与流程对比）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-10 |
| 场景/触发条件 | 招标办表示"另一家投标方的资质和过往业绩看起来更齐全" |
| 对话对象角色 | 招标办/采购流程负责人 |
| 标准话术 | "我们的资质材料已经按招标文件要求逐项提供，如果有具体缺项可以现在补充说明。另外我们可以提供近三年同类项目的验收报告原件供核实，确保业绩真实可追溯。" |
| 禁忌话术 | 不要笼统说"我们资质没问题"却不逐项核对招标文件的具体要求；不要质疑评审流程本身 |
| 话术意图（四维度） | **情绪**：从"资质存疑"的警惕，转为"逐项可核实"的踏实；**信息目标**：明确招标办具体认为哪项资质或业绩存在不足；**关系**：维护招投标流程的公平性和自身合规形象；**行动导向**：促成招标办认可资质完整性，进入下一评审环节 |
| 升级路径 | 若资质或流程疑虑经补充材料仍未消除：转交项目负责人正式回函，进入书面澄清程序，不再通过口头沟通处理 |
| 使用注意 | 需提前逐条比对招标文件的资质要求清单 |

**角色分支2：技术把关人（关注技术方案对比，非合规）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-11 |
| 场景/触发条件 | 技术把关人说"另一家的方案参数看起来比你们的更先进" |
| 对话对象角色 | 技术把关人 |
| 标准话术 | "参数只是方案的一部分，更重要的是参数和您现场实际工况的匹配度。我们的方案是基于现场勘测数据定制的，可以逐项对比：哪些参数是贵司实际需要的，哪些可能是对方为了数字好看堆上去、但用不到的。" |
| 禁忌话术 | 不要单纯比拼参数数字本身；不要否认对方参数数据的真实性 |
| 话术意图（四维度） | **情绪**：从"我们参数不如人"的被动，转为"参数匹配需求才是关键"的主导；**信息目标**：了解技术把关人具体在意哪几项参数指标；**关系**：建立"以需求为中心"而非"以参数竞赛"的专业形象；**行动导向**：促成技术把关人认可"匹配度"这一评估标准 |
| 升级路径 | 若技术把关人经数据对比仍坚持参数优先：安排技术把关人与自家技术团队现场对话，转交技术负责人处理，不由销售继续争论参数本身 |
| 使用注意 | 需提前准备好现场勘测数据与竞品公开参数的对比材料 |

> 说明：招标办关注合规与流程完整性，技术把关人关注技术方案是否匹配实际需求，两者关注点实质不同，符合角色分支规则，予以拆分。

### 异议类型：预算/决策延迟类

**子场景1：决策未定（内部讨论中）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-12 |
| 场景/触发条件 | "我们内部还要再讨论，具体时间不好说" |
| 对话对象角色 | 分管领导/技术把关人（均可能提出，视沟通对象而定；若关注点无实质差异，不做角色拆分） |
| 标准话术 | "理解，大项目决策慎重是应该的。方便问一下，讨论主要还剩哪个环节，是技术评估还是预算审批？我们这边可以针对性补充材料，加快评审进度。" |
| 禁忌话术 | 不要频繁催促决策进度；不要在决策链未走完前跳过流程直接找更高层施压 |
| 话术意图（四维度） | **情绪**：从"被无限期拖延"的焦虑，转为"了解具体卡点、可针对性推进"的主动感；**信息目标**：明确剩余决策环节具体卡在哪一步；**关系**：不破坏与招标流程/决策链的正常节奏，避免留下"不专业"的印象；**行动导向**：针对具体卡点补充材料，推进流程进入下一环节 |
| 升级路径 | 若持续多轮跟进仍无实质进展：转入S14长周期跟进节奏表的常规节奏管理，不再作为单独异议反复处理 |
| 使用注意 | 乙决策周期长（3-9个月），此类异议需要耐心多轮跟进，不可急于求成 |

**子场景2：流程堵塞（决策已启动但卡在执行环节，新增）**

| 字段 | 内容 |
|---|---|
| 编号 | S11-13 |
| 场景/触发条件 | "方案已经提交财务审批了，但对方部门一直没反馈，也联系不上具体进度" |
| 对话对象角色 | 项目对接人（非决策角色本身，而是流程协调对象） |
| 标准话术 | "这种情况理解，跨部门流程有时候确实会卡住。方便帮忙问一下财务那边具体卡在哪个环节吗？如果需要，我们也可以配合提供补充材料，加快审批。" |
| 禁忌话术 | 不要越级直接联系财务施压；不要表现出不耐烦或频繁催促对接人 |
| 话术意图（四维度） | **情绪**：从"被无限拖延"的焦虑，转为"共同协调推进"的合作感；**信息目标**：明确流程卡在哪个具体环节，而非笼统的"还没批"；**关系**：维护与对接人的协作关系，不因催办而破坏信任；**行动导向**：促成对接人主动去了解或推动财务审批进度 |
| 升级路径 | 若连续两周无进展、且对接人也无法推进：可请项目负责人出面，通过更高层级的关系协调，但需谨慎评估是否会被视为越级施压 |
| 使用注意 | 与"内部还要再讨论"场景的区别：这里是决策已启动但流程卡住，不是决策本身未定，应对逻辑是"协调推进"而非"澄清顾虑" |

### 多角色同时发难：评审会应对策略（新增）

乙的方案评审会场景中，技术把关人、财务、招标办等角色经常在同一场会议上先后或同时提出不同类异议（如技术质疑配置冗余，财务同时质疑总价），此时不能孤立套用单一角色的话术，需要按顺序穿插应对：

| 环节 | 处理策略 |
|---|---|
| 识别开场 | 先快速判断在场角色分别提出的是哪类异议（可参考S11-07技术把关人-价格类 / S11-06财务-价格类 的判断依据），不要一收到多个问题就同时展开回应 |
| 应对顺序 | 优先回应技术把关人的疑虑（技术可行性是财务愿不愿意批预算的前提），技术疑虑未消除前，财务的成本问题往往难以真正解决；技术认可后，再转向财务的总成本论证（参照S11-06升级路径） |
| 穿插话术 | 用过渡语衔接不同角色的关注点，如"刚才和XX确认了配置的必要性，那我们再看一下整体投入产出" ，避免让在场人员感觉在被分别"打发" |
| 升级路径 | 若会议现场无法当场消除全部疑虑：不勉强当场拍板，明确会后提供哪些补充材料（技术仿真报告/成本核算说明），分别对应S11-07、S11-06的升级路径，约定具体反馈时间 |
| 使用注意 | 现场应对多角色时，语速放慢、逐一回应，避免因追求"一次说清楚"而让话术显得杂乱、降低专业感 |

## 四、本模块专属验收细则（在通用验收标准框架基础上叠加）

| 检查点 | 标准 |
|---|---|
| 前置识别指南是否包含 | 是否包含模糊表述对照表和复合异议处理规则，不能直接跳到分类话术 |
| 异议类型覆盖 | 甲乙两边均覆盖4类高频异议（价格/竞品对比/预算延迟/信任风险），不能只写1-2类 |
| 甲乙差异体现 | 同一异议类型下，甲乙话术是否体现出决策速度、决策人角色、证据形式（口头vs书面案例）的实质差异 |
| 角色分支合理性 | 角色拆分是否符合"关注点实质不同才拆分"的规则，是否为不拆分的情况给出明确说明依据 |
| 标准/禁忌话术对照完整 | 每条异议必须同时有"该怎么说"和"不能怎么说" |
| 话术意图四维度完整 | 情绪、信息目标、关系、行动导向四项均需填写，不能只写一两项 |
| 升级路径明确 | 每条异议是否说明二轮应对方向或转交条件，不能空缺或只写"继续沟通" |
| 编号完整且唯一 | 每条正文条目是否按{模块编号}-{两位序号}格式编号，且无重复、无跳号 |

## 五、课件页要点（讲解逻辑，非逐字稿）

1. 先讲异议识别的通用原则：**异议不是拒绝，是需要更多信息的信号**；客户表述常模糊或复合，先澄清再套话术，不要见关键词就直接背话术。
2. 展示甲乙同一异议类型的话术对比，让学员直观看到"业务模式不同，应对逻辑也不同"。
3. 展示同一案例内部的角色分支对比（如甲的采购经理版vs老板版），让学员理解"面对不同角色，同一异议的应对重点会变"。
4. 讲清楚"升级路径"的意义：**标准话术只是第一轮，客户不接受是常态**，一线销售需要清楚自己的权限边界，知道什么时候该继续争取、什么时候该转交，而不是硬扛到底或轻易许诺超出权限的条件。
5. 现场练习：给学员一个本企业真实遇到过的异议场景（先要求学员自行判断属于哪类异议、是否复合），仿照模板结构（触发条件→角色→标准话术→禁忌话术→四维度意图→升级路径）当场填一版，导师用本模块验收细则现场点评。

# S07 拜访前准备清单

> 版本：v1.0（正式稿）
> 篇章：五、售前筹备篇
> 模板类型：类型C2（清单目录型）
> 适用业务模式：甲乙差异明显（乙的准备复杂度远高于甲）
> 前置依赖：字段规范文档 v1.8、案例公司设定文档、S05决策链画像与分角色沟通策略
> 模块目的：让每次拜访都有备而来，避免因准备不足错失机会或显得不专业

---

## 一、模块范围说明（重要）

25个正文模块的篇章顺序本身即按客户旅程阶段组织（客户画像→获客拓客→售前筹备→需求挖掘→方案与报价→异议处理→成交……），**本模块特指初次拜访前的准备**，对应"售前筹备篇"在整体顺序中的位置。

后续阶段的会议/拜访准备**不在本模块覆盖范围内**，而是分散在对应阶段的模块中：需求挖掘/方案讲解会前的准备见S08、S09；报价谈判/临门一脚前的准备见S10、S13。这样设计是为了避免与整体篇章结构重复冗余，而非遗漏。

## 二、案例公司甲（标准化产品/中小企业客户）拜访前准备清单

| 条目名称 | 具体做法 | 责任人/角色 | 产出物/检查标准 |
|---|---|---|---|
| 客户背景资料 | 提前了解客户企业规模、主营产品、大致采购品类，可通过企查查、客户官网了解 | 拜访销售 | 能说出客户基本经营情况，而非空白拜访 |
| 报价/产品资料 | 携带标准报价单、产品参数手册、样品目录 | 拜访销售 | 资料齐全，格式统一（参照公司标准模板） |
| 产品样品 | 根据客户可能感兴趣的品类携带对应样品 | 拜访销售 | 样品与客户实际需求品类匹配 |
| 沟通目标设定 | 明确本次拜访要达成的具体目标（如获取采购计划、推进小批量试单），而非"去聊聊看" | 拜访销售 | 拜访前能清楚说出本次目标是什么 |
| 客户痛点预判 | 结合S03客户画像和以往接触记录，预判客户可能关心的问题（价格、交期、质量） | 拜访销售 | 能提前准备好对应S02、S11相关话术 |
| 小礼品（视情况） | 根据客户关系阶段决定是否携带轻量级礼品（如公司定制文具） | 拜访销售 | 礼品符合企业合规规定，不涉及商业贿赂风险 |
| 个人形象与状态准备 | 参照S26销售个人可信度塑造模块 | 拜访销售 | 参照S26对应验收标准 |

## 三、案例公司乙（定制化项目/大企业政府客户）拜访前准备清单

| 条目名称 | 具体做法 | 责任人/角色 | 产出物/检查标准 |
|---|---|---|---|
| 客户背景与决策链资料 | 提前了解客户所属机构性质、以往合作案例（如有）、结合S05方法预判可能的决策链角色 | 项目负责人 | 能预判本次拜访大致会接触到哪些角色 |
| 资质与合规材料 | 携带企业资质证书、过往同类项目验收报告复印件/电子版 | 商务负责人 | 材料齐全且为最新有效版本，避免资质过期 |
| 方案/技术资料 | 携带初步方案框架或技术能力介绍材料（非正式报价，视拜访阶段而定） | 项目负责人/技术负责人 | 材料内容与客户前期沟通的需求匹配，非通用模板照搬 |
| 沟通目标设定 | 明确本次拜访要达成的具体目标（如摸清决策链、推进技术交流、确认招标节点） | 项目负责人 | 拜访前能清楚说出本次目标，且符合当前项目阶段 |
| 客户顾虑预判 | 结合S05角色关注点，预判本次接触对象最可能关心的问题类型 | 项目负责人 | 能提前准备好对应角色的S11话术 |
| 同行人员安排 | 视拜访对象决定是否需要技术负责人/公司高层陪同（如涉及技术把关人或分管领导） | 项目负责人 | 陪同人员角色与拜访对象匹配（如技术把关人对接需技术负责人陪同） |
| 多人拜访分工准备 | 若安排多人同行，需提前明确主谈人（主导话题推进）和观察人（负责倾听、观察在场其他人反应、记录关键信息）的角色分工，避免现场抢话或都不主动开口 | 项目负责人 | 出发前已明确谁主谈、谁观察记录，且陪同人员知晓自己的角色 |
| 个人形象与状态准备 | 参照S26销售个人可信度塑造模块 | 全体拜访人员 | 参照S26对应验收标准 |

## 四、拜访结束后的信息记录衔接（通用规则）

拜访结束后24小时内，应将本次收集到的决策链相关信息（如新识别的角色、态度变化、顾虑内容）更新至S05决策链地图模板，避免信息遗失、下次拜访又从零开始了解客户情况。此项由拜访销售/项目负责人在拜访结束后自行完成，不依赖他人提醒。

## 五、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| 条目覆盖完整 | 是否覆盖资料、样品/方案、沟通目标、顾虑预判、个人形象等主要准备类别 |
| 产出标准可检验 | 每项准备是否有明确的完成判断标准，而非"准备好了"这种模糊说法 |
| 甲乙差异体现 | 乙的准备是否体现资质合规、决策链预判、同行人员安排及分工这些甲不需要考虑的复杂度 |
| 多人分工是否明确 | 乙的多人拜访场景是否提前明确主谈人和观察人角色，而非现场临时协调 |
| 拜访后信息记录衔接是否存在 | 是否说明拜访后信息该更新到哪里，避免与S05决策链地图脱节 |

## 六、课件页要点

1. 强调"沟通目标设定"是最容易被忽略但最重要的一项——很多销售出发前没想清楚这次到底要达成什么，导致拜访变成漫无目的的聊天。
2. 讲清楚本模块范围定位：只覆盖初次拜访，后续阶段的准备在对应模块（S08/S09/S10/S13）里各自处理，避免学员误以为S07该覆盖所有拜访场景。
3. 讲解乙的多人拜访分工，用一个"两人都抢着说话"或"两人都沉默等对方开口"的反面案例说明提前分工的必要性。
4. 强调拜访后的信息记录习惯，让学员理解这不是额外负担，而是让下次拜访更高效的必要动作。
5. 现场练习：给学员一个即将到来的初次拜访场景，让其按清单逐项列出准备内容（含个人形象、多人分工），导师检查是否遗漏关键项。

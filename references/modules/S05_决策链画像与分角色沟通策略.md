# S05 决策链画像与分角色沟通策略

> 版本：v1.0（正式稿）
> 篇章：三、客户画像篇
> 模板类型：混合（类型C识别流程 + 类型A角色沟通策略）
> 适用业务模式：甲乙对比（乙尤其关键，决策链复杂度是甲的数倍）
> 前置依赖：S03（乙类客户决策权限待确认时的摸底方法）、S11（各角色异议应对话术）、字段规范文档 v1.7
> 模块目的：让销售（尤其面对乙类客户时）能识别谁是真正的决策人、该对谁说什么，避免"一套话术打天下"导致谈不到点上

---

## 一、决策链角色识别方法

**甲（决策链简单，识别难度低）**

| 识别线索 | 判断依据 |
|---|---|
| 谁在报价/合同环节被反复提及 | 通常是采购经理，日常决策的实际执行人 |
| 大额订单是否被提及需要"再确认"或"过一下" | 暗示存在老板审批环节，需主动询问审批流程和权限门槛 |

**乙（决策链复杂，需要主动摸排）**

| 识别线索 | 判断依据 |
|---|---|
| 谁负责组织现场勘测、提供技术参数 | 通常是技术把关人 |
| 谁提及"预算""财务""走流程" | 通常指向财务/预算审批方 |
| 谁提及"招标文件""资质要求""评审流程" | 通常是招标办/采购流程负责人 |
| 谁在最终决策节点被提及为"还要报给XX" | 通常是分管领导，是否为最终拍板人需要进一步确认 |
| 是否提及"专家评审""第三方评估" | 存在第三方评审专家，通常在方案技术评审环节介入 |

**识别提问模板**（可在拜访前准备或需求挖掘阶段自然带出）：
- "这个项目除了您这边，还有哪些部门会参与评估？"
- "最终这个决策大概是什么流程，会经过哪几个环节？"
- "如果我们方案通过了技术评审，接下来是您这边决定还是需要往上再报？"

## 二、角色影响力优先级判断（角色间意见分歧时该怎么办）

不同角色的实际影响力并不均等，需先区分角色类型，再决定分歧出现时该优先攻克谁：

| 判断维度 | 说明 |
|---|---|
| 否决型角色 vs 影响型角色 | 财务、招标办通常是"否决型"（预算不批/资质不合规，项目直接终止，无商量余地）；技术把关人、分管领导更多是"影响型"（意见很重要，但不直接卡死流程） |
| 分歧处理优先顺序 | 多个角色意见分歧时，**优先解决否决型角色的疑虑**（不解决意味着流程根本无法推进），影响型角色的支持可作为说服否决型角色的佐证材料 |
| 借力策略 | 争取到强影响力角色（技术把关人、分管领导）的正面意见后，应尽量转化为**书面材料**（邮件确认、评审意见）用于说服其他角色，而非仅口头转达，书面材料更有说服力且不易失真 |
| 升级仲裁 | 若中层角色（如技术把关人与财务）意见持续分歧且无法自行调和，可通过项目负责人请求分管领导出面裁决；但需谨慎评估是否构成越级，一般仅在项目停滞较长时间、常规沟通无效后才采用 |

## 三、关键角色接触受限时的应对策略

现实中并非所有已识别的关键角色都愿意或方便直接接触，尤其乙的高层角色常有意保持距离：

| 情况 | 应对方式 |
|---|---|
| 分管领导/决策层不直接接触，只通过下属传话 | 通过书面材料（方案摘要、案例背书）传递关键信息，确保下属转达时不失真；借助内部教练（见第四节）侧面了解态度反馈 |
| 完全没有渠道触达关键角色 | 寻找间接接触机会（如项目里程碑节点的汇报会、答谢活动、行业会议），或通过更高层级关系协调安排一次正式引荐 |
| 使用注意 | 不强行越级直接联系，避免破坏现有对接关系；以间接影响为主，不代替对该角色真实态度的判断，仍需通过其他渠道验证间接传达信息的准确性 |

## 四、内部教练识别与培养

**内部教练**指客户内部愿意主动提供信息、协助我方了解决策链真实情况的人，通常不是最终决策人，但对项目推进有重要信息价值。

| 项目 | 说明 |
|---|---|
| 识别信号 | 主动透露决策链内部信息（如"其实真正拍板的是XX"）；主动提醒我方注意某些环节或竞品动态；对我方方案表现出超出职责范围的支持热情 |
| 培养方式 | 保持对该角色的额外关注和及时反馈（如让其感觉自己的建议被重视）；不越权索取其无法提供的敏感信息，避免让其为难 |
| 使用注意 | **不能只依赖单一内部教练的信息**，其视角可能片面或存在部门利益倾向，需与识别清单中的其他线索交叉验证；不能让内部教练在客户内部因为过度帮我方而承担风险 |

## 五、分角色沟通策略（正文）

### 案例公司甲

**S05-01｜采购经理**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 价格、交期、参数细节、个人考核压力（成本控制指标） |
| 信息呈现优先级 | 价格对比数据 > 交期保障细节 > 长期合作案例 |
| 建立信任关键动作 | 快速响应询价、提供清晰的参数对比材料，展现专业和高效 |
| 常见地雷 | 不要越过采购经理直接联系老板谈价格，会破坏对接关系；不要用对老板讲的"战略价值"话术对采购经理讲，其KPI导向不同 |
| 关联话术 | 参照S11-01（价格类异议）、S02-01（交期卖点） |

**S05-02｜老板/总经理**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 供应链稳定性、长期合作价值、企业整体经营风险 |
| 信息呈现优先级 | 长期合作价值和风险规避 > 战略性框架条款 > 具体参数细节 |
| 建立信任关键动作 | 展现对客户整体经营的理解而非单笔生意视角，体现"伙伴"而非"供应商"定位 |
| 常见地雷 | 不要对老板堆砌参数细节，会显得不懂决策层沟通逻辑；不要绕过采购经理直接频繁联系老板，会破坏内部关系 |
| 关联话术 | 参照S11-02（价格类异议老板版）、S01-03（10分钟版企业介绍，适合决策层） |

### 案例公司乙

**S05-03｜技术把关人**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 技术可行性、方案是否匹配实际工况、是否存在技术风险 |
| 信息呈现优先级 | 现场勘测数据和技术验证 > 同类案例佐证 > 商务条款 |
| 建立信任关键动作 | 提供具体、可验证的技术材料（仿真报告、勘测记录），邀请与自家技术团队直接对话 |
| 常见地雷 | 不要用商务话术（价格优惠、合作意向）回应技术疑虑，答非所问会失去专业信任；不要在技术细节上含糊搪塞 |
| 关联话术 | 参照S11-07（价格类异议技术把关人版）、S11-09（信任风险技术把关人版）、S11-11（竞品对比技术把关人版） |

**S05-04｜财务/预算审批方**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 总成本合理性、预算是否在授权范围内、付款条款风险 |
| 信息呈现优先级 | 全生命周期成本数据 > 付款条款灵活性 > 具体技术细节 |
| 建立信任关键动作 | 提供书面、可核算的成本材料，而非口头说服 |
| 常见地雷 | 不要只谈"性价比"这种模糊表述，财务需要具体数字；不要在评审会现场承诺任何未经审批的让价 |
| 关联话术 | 参照S11-06（价格类异议财务版） |

**S05-05｜招标办/采购流程负责人**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 流程合规性、资质完整性、评审公平性 |
| 信息呈现优先级 | 资质材料完整度 > 过往业绩可验证性 > 商务条款 |
| 建立信任关键动作 | 严格按招标文件逐项提供材料，展现流程配合度和专业度 |
| 常见地雷 | 不要试图绕过流程走"关系路线"，容易引发合规风险；不要质疑评审流程本身，即使认为流程对己方不利 |
| 关联话术 | 参照S11-10（竞品对比招标办版） |

**S05-06｜分管领导**

| 字段 | 内容 |
|---|---|
| 角色关注点 | 决策责任风险、项目整体成功率、是否有可追责的保障机制 |
| 信息呈现优先级 | 可验证的案例背书和责任条款 > 技术细节 > 价格 |
| 建立信任关键动作 | 提供可核实的同类项目案例和对接人联系方式，让分管领导能自行验证而非只听我方一面之词 |
| 常见地雷 | 不要口头承诺无限责任；不要含糊回应经验问题却拿不出具体案例 |
| 关联话术 | 参照S11-08（信任风险分管领导版） |

## 六、决策链地图绘制模板（用于记录、跟踪单个客户的决策链现状）

| 角色 | 姓名/职位 | 对我方态度（支持/中立/反对/未知） | 关键顾虑 | 是否为内部教练 | 识别置信度（高/中/低） | 最近一次接触时间 |
|---|---|---|---|---|---|---|
| （按识别到的角色逐行填写，乙类客户通常需要填4-6行） | | | | | | |

**使用说明**：
- 此表应在每次拜访/沟通后更新，尤其"对我方态度"和"关键顾虑"两栏，用于判断该客户整体决策链的支持度分布，避免只关注单一联系人而忽略了其他关键角色的态度变化。
- **识别置信度**字段用于标记该角色定位的把握程度：置信度为"低"或"中"的角色，应优先安排下一次接触加以验证，不能长期基于未经验证的判断制定沟通策略。
- **若发现某角色实际影响力与最初判断严重不符**（如误判技术把关人为最终拍板人，后发现另有决策人主导），应立即更新地图并调整沟通资源分配，不因"已经投入很多精力说服原判断对象"而固守原策略——沉没成本不应成为继续误判下去的理由。

## 七、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| 识别方法可操作 | 识别线索和提问模板是否具体，能否指导新人现场判断角色 |
| 每个角色策略完整 | 是否包含关注点、信息优先级、信任建立动作、常见地雷四项，不能只写关注点 |
| 与S11话术关联准确 | 关联的S11编号是否与该角色实际对应的话术条目一致 |
| 角色影响力优先级是否明确 | 是否区分否决型/影响型角色，并说明分歧时的处理顺序 |
| 接触受限应对是否覆盖 | 是否提供关键角色难以直接触达时的间接影响策略 |
| 内部教练使用边界清楚 | 是否说明不能只依赖单一线人、不能让线人承担过度风险 |
| 决策链地图可用 | 模板是否具备可持续更新、追踪态度变化及识别置信度的实用性，而非一次性静态记录 |

## 八、课件页要点

1. 强调"一套话术打天下"的失败根源：不同角色关注点本质不同，用错误的信息优先级去说服错误的对象，是B2B销售最常见的低级错误。
2. 讲解角色影响力优先级：分歧出现时先啃否决型角色，用影响型角色的支持作为佐证材料，而非平均用力说服所有人。
3. 讲解关键角色接触受限时如何间接施加影响，避免因为"见不到人"就放弃该角色的工作。
4. 讲解内部教练的识别与培养，同时强调不能形成信息依赖单一来源的风险。
5. 强调决策链地图不是一次性画完就结束，识别置信度低的角色要主动安排验证，发现误判要及时调整、不因沉没成本固守错误判断。
6. 现场练习：给学员一个乙类客户的模拟场景描述，让其用识别方法判断决策链结构、画出决策链地图（含识别置信度），并说明该对每个角色采取什么沟通策略、遇到分歧时优先解决谁。

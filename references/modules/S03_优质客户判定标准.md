# S03 优质客户判定标准

> 版本：v1.0（正式稿）
> 篇章：三、客户画像篇
> 模板类型：类型B（判定标准）
> 适用业务模式：甲乙对比（判定维度权重不同）
> 前置依赖：字段规范文档 v1.6、案例公司设定文档
> 模块目的：让销售把时间和资源投入到值得投入的客户身上，避免凭感觉判断浪费在低价值客户上
> 与S04的关系：S03解决"要不要跟进"（合格与否的门槛判断），S04解决"跟进到什么程度"（已确定值得跟进后的资源分配），S03在前、S04在后

---

## 一、判定维度总览（甲乙通用维度，权重不同）

| 判定维度 | 说明 |
|---|---|
| 需求匹配度 | 客户当前需求是否与我方产品/服务能力真实匹配 |
| 决策权限清晰度 | 接触的对象是否有实际决策权或能接触到决策人 |
| 支付能力与信用 | 客户是否有稳定的支付能力，是否有不良信用记录 |
| 合作诚意 | 客户是否只是"询价比价"，还是有真实合作意愿 |
| 成长潜力 | 客户未来是否有复购、增购或转介绍的可能性 |

---

## 二、三种判定状态的区分（重要前提）

每个维度的判定结果需区分以下三种状态，处理方式不同，不能混用：

| 状态 | 含义 | 处理方向 |
|---|---|---|
| 合格/清晰 | 有明确正面信号，符合标准 | 正常推进 |
| **存疑** | 有明确的负面信号（如信用不良记录、陪标迹象） | 需谨慎处理或转交主管评估风险 |
| **信息不足** | 目前接触次数/信息量不够，无法判断是正面还是负面（不是负面信号，只是还不知道） | 暂缓判定，通过后续接触补充信息，不直接转交主管，也不因此放弃跟进 |

此前版本容易将"存疑"和"信息不足"混为一谈，导致信息不足时也被当作风险处理、过早转交主管或过早放弃投入。两者的关键区别：存疑是"知道了、结果不好"，信息不足是"还不知道"。

## 三、案例公司甲（标准化产品/中小企业客户）判定标准

| 判定维度 | 判定取值/标准 | 对应状态/结论 | 该结论对应动作 |
|---|---|---|---|
| 需求匹配度 | 采购品类与我方产品高度匹配，采购量在常规供货范围内 | 合格 | 进入正式跟进流程 |
| | 采购品类部分匹配，或采购量远低于最小起订量 | 部分匹配 | 评估是否值得投入资源，若能满足最小起订量以上可继续 |
| | 采购品类完全不符（如需要我方不生产的规格） | 不匹配（一票否决） | 礼貌说明不在服务范围，不投入跟进资源 |
| 决策权限清晰度 | 直接对接采购经理或以上，可独立决策或明确知道审批流程 | 清晰 | 正常推进 |
| | 对接人为基层人员，尚未有机会确认其决策链角色 | 信息不足 | 下次接触时主动询问对接人在决策链中的角色，暂不因此减少常规投入 |
| 支付能力与信用 | 企业经营正常，无公开负面信用记录 | 合格 | 正常推进 |
| | 存在信用风险信号（如同行反馈拖欠货款、经营异常） | 存疑 | 转交主管评估，考虑现款现货或缩小首单规模降低风险 |
| | 尚未了解客户经营状况（初次接触，无从判断） | 信息不足 | 通过企查查等公开渠道或后续接触了解基本经营信息，暂不转交主管 |
| 合作诚意 | 主动提供采购计划、愿意讨论具体细节 | 真实意向 | 正常推进 |
| | 只反复要求报价、不透露采购计划或时间表 | **疑似比价** | 可提供简要报价，但不投入深度方案设计资源。**重新评估触发条件**：若客户后续主动提供采购计划、时间表，或明确表示要推进合作细节，立即按"真实意向"重新判定，恢复正常资源投入，不因之前的"疑似比价"标签而持续冷处理 |
| 成长潜力 | 企业处于扩张期，或有明确的持续采购计划 | 潜力较高 | 优先投入资源，纳入重点培养对象 |
| | 一次性/临时采购需求，无持续合作迹象 | 潜力一般 | 正常服务，不额外投入资源 |

**综合判定规则**：
1. **一票否决**：若"需求匹配度"为不匹配，或"支付能力与信用"判定为存疑（非信息不足），直接判定为不值得跟进/需谨慎处理，不看其他维度。
2. **多维度信息不足的处理**：若一票否决维度均无负面信号，但其余维度中有一半以上处于"信息不足"状态（既非合格也非存疑），按"低资源投入观察"处理——维持常规联系节奏，不投入深度方案设计或大幅让价资源，同时在后续每次接触中主动补充缺失信息，信息补齐后重新按本标准判定。

## 四、案例公司乙（定制化项目/大企业政府客户）判定标准

| 判定维度 | 判定取值/标准 | 对应状态/结论 | 该结论对应动作 |
|---|---|---|---|
| 需求匹配度 | 项目需求与我方技术能力和过往案例高度匹配 | 合格 | 进入正式跟进流程 |
| | 项目需求超出我方技术能力范围，或规模远超/远低于我方项目承接能力 | 不匹配（一票否决） | 评估是否需要引入合作伙伴，或礼貌说明能力边界 |
| 决策权限清晰度 | 已接触到技术把关人、财务或分管领导中至少一位，且能确认招投标流程节点 | 清晰 | 正常推进 |
| | 仅接触到基层联络人，尚未有机会摸清完整决策链 | 信息不足 | 通过S05决策链画像方法，尝试摸清完整决策链，不因此立即判定为不合格或转交主管 |
| 支付能力与信用 | 为政府机构或大型企业集团，具备正常财政/资金能力 | 合格 | 正常推进 |
| | 项目资金来源不明确，或存在历史拖欠工程款先例 | 存疑 | 转交公司高层评估，考虑分期付款条款或前期风险规避条款 |
| | 尚未了解项目资金来源（初次接触阶段） | 信息不足 | 在方案沟通过程中逐步了解资金来源和预算流程，暂不转交高层 |
| 合作诚意 | 已进入正式招投标流程，或明确表达非公开招标的合作意向 | 真实意向 | 正常推进，投入方案设计资源 |
| | 仅索要资料用于"货比三家"或陪标凑数，无实质推进迹象 | **疑似陪标** | 提供基础资料应付流程要求，不投入实质性方案设计资源。**重新评估触发条件**：若客户后续正式进入招投标流程、发出正式邀标函，或安排现场勘测等实质性动作，立即按"真实意向"重新判定，恢复资源投入 |
| 成长潜力 | 属于长期战略客户类型（如产业园区、集团总部级项目），后续可能有多期合作 | 潜力较高 | 优先投入资源，考虑高层直接介入 |
| | 一次性项目，后续无明确延续可能 | 潜力一般 | 正常投入，不额外倾斜资源 |

**综合判定规则**：
1. **一票否决**：与甲相同逻辑（需求匹配度不匹配、支付能力存疑即不跟进/谨慎处理）。
2. **决策权限的特殊处理**：与甲不同，"决策权限清晰度"若为"信息不足"，不直接判定不跟进，而是先通过S05摸清决策链再决定——这是甲乙判定流程上的关键差异，乙需要先做决策链摸底这一步，甲可以较快下判断。
3. **多维度信息不足的处理**：与甲相同的"低资源投入观察"逻辑，但由于乙决策周期本身较长，观察期可适当放宽，不必急于在短期内强行判定。

## 五、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| 一票否决维度明确 | 是否清楚说明哪些维度不合格时直接判定不跟进，不能所有维度都当作可综合权衡 |
| 判定取值可操作 | 每个取值是否有具体可判断的标准，而非"看起来还不错"这种主观描述 |
| 每个结论配有动作 | 是否每个判定结果都对应具体该怎么做，而非只给结论不给动作 |
| 存疑与信息不足是否区分 | 是否清楚区分"有负面信号"和"信息不够"这两种不同状态，处理方式是否对应不同 |
| 重新评估路径是否存在 | "疑似比价""疑似陪标"等负面标签是否给出了后续可能转变、重新判定的触发条件，而非一次贴标签就定型 |
| 多维度模糊场景是否覆盖 | 是否说明当多个维度同时处于信息不足状态时该如何处理，而非只覆盖单一维度不合格的清晰情况 |
| 甲乙差异体现 | 是否体现乙需要先摸清决策链、甲可直接判断这一本质差异 |

## 六、课件页要点

1. 讲清楚"优质客户判定"和"客户分层"的区别：S03是"要不要投入"，S04是"投入多少"，先过S03这道门槛，再进入S04的分层管理。
2. 强调一票否决逻辑：需求不匹配或支付能力存疑的客户，不该因为其他维度看起来不错就勉强跟进。
3. 讲清楚"存疑"和"信息不足"的区别，避免学员把"还不了解"当成"有问题"，过早放弃或过度谨慎。
4. 强调"标签不是终身制"：疑似比价、疑似陪标的客户，一旦出现实质性转变信号，应立即重新评估，不要被最初的判断锁死。
5. 现场练习：让学员用本模块标准，对3-5个真实客户案例打分判定，其中至少设计一个"多维度信息不足"的模糊案例，导师用验收细则点评。

# S02 产品BEDS话术

> 版本：v1.0（正式稿）
> 篇章：二、产品价值篇
> 模板类型：类型A（话术库）
> 适用业务模式：通用（以案例公司甲为示范，乙可参照替换背景信息）
> 适用客户分层：不区分
> 前置依赖：字段规范文档 v1.6、案例公司设定文档
> 模块目的：让销售讲产品时始终围绕"客户利益"展开，而非自说自话罗列参数，统一产品价值表达口径

---

## 一、BEDS方法论说明

| 要素 | 含义 | 常见误区 |
|---|---|---|
| B 背景 | 这个卖点解决的是客户什么场景下的什么问题 | 常见误区：跳过背景直接讲优势，客户不知道"这跟我有什么关系" |
| E 优势 | 我们在这一点上具体做了什么、有什么不同 | 常见误区：优势描述空泛（"质量好""服务好"），没有具体做法支撑 |
| D 证明 | 用什么可验证的事实证明这个优势是真的 | 常见误区：只有说法没有证据，客户将信将疑 |
| S 利益 | 这个优势最终能给客户带来什么实际好处 | 常见误区：讲完证明就结束，没有翻译成客户能感知的具体利益 |

**说明**：本模块示范2个核心卖点（交期稳定性、质检可追溯性），企业实际使用时可能有更多核心卖点（如批量折扣、定制化包装方案等），均可按BEDS四要素结构自行补充，不局限于本模块示范数量。

---

## 二、卖点选择依据（初次接触时该先讲哪个）

客户初次接触时往往不会主动说明自己最在意哪个维度，销售需要先判断优先带出哪个卖点，而非把所有卖点一次性倒出。

| 客户初期信号 | 优先带出的卖点 |
|---|---|
| 主动提及"之前交期总是delay""排产紧张" | 优先讲S02-01交期稳定性 |
| 主动提及"质量不稳定""出过问题不好追溯" | 优先讲S02-02质检可追溯性 |
| 未主动提及具体顾虑（初次接触，尚不了解客户痛点） | 先按通用顺序讲交期稳定性（多数中小企业采购最关心的痛点），再视客户反应决定是否深入质检话题，不必两个都讲全 |

## 三、正文

### S02-01｜核心卖点：交期稳定性

| 字段 | 内容 |
|---|---|
| 场景/触发条件 | 客户在了解产品阶段，尚未进入价格谈判，销售主动介绍核心卖点 |
| 对话对象角色 | 采购经理 |
| 标准话术 | **B（背景）**："很多客户之前跟我们合作前，最头疼的是供应商交期不稳定，紧急订单经常延误影响自己的生产排期。" **E（优势）**："我们专门为紧急订单开辟了72小时补货通道，同时保持核心型号常备库存。" **D（证明）**："过去一年我们对XX、XX这两家客户的紧急订单响应记录都在72小时内完成，可以提供交付记录供参考。" **S（利益）**："这意味着您这边即使遇到突发排产需求，也不用担心因为我们这边延误而打乱您的生产计划。" |
| 禁忌话术 | 不要跳过B直接说"我们交期很快"，客户没有代入感；不要在D阶段只说"我们服务很好"这种无法验证的话，必须给可查的具体记录 |
| 话术意图（四维度） | **情绪**：从"对交期的担忧"转为"这个供应商能解决我的痛点"的信任感；**信息目标**：了解客户过去是否真的遇到过交期问题、影响程度多大；**关系**：建立"我们理解你的实际困扰"而非"自卖自夸"的专业形象；**行动导向**：引导客户主动询问具体的交期保障细节或提出自己的排产需求 |
| 升级路径 | 若客户对交期不太关心（更看重价格）：简要带过，转向S02-02质检可追溯性或直接进入需求挖掘环节（对应后续模块，待模块产出后补充具体编号引用）；若客户追问具体细节：可展示交付记录或转入方案设计环节说明具体保障方案（对应后续模块，待模块产出后补充具体编号引用） |
| 使用注意 | D阶段的证明材料需要是真实可查的，不能用编造的案例数字，一旦被客户核实发现不实，会彻底损害信任 |

### S02-02｜核心卖点：质检可追溯性

| 字段 | 内容 |
|---|---|
| 场景/触发条件 | 客户对产品质量稳定性有顾虑，或主动询问质检流程 |
| 对话对象角色 | 采购经理或技术对接人 |
| 标准话术 | **B（背景）**："不少客户反映之前用过的供应商，出了质量问题很难追溯是哪个批次、哪个环节的问题。" **E（优势）**："我们每个批次都有独立的质检报告和批次编号，出厂前经过三道质检工序。" **D（证明）**："这是我们标准的质检报告模板，每批货都会随货附上，您可以看到具体的检测项目和数值。" **S（利益）**："如果未来使用过程中出现任何问题，可以直接根据批次编号快速定位原因，不会出现说不清楚责任的情况，也能帮您更快向下游客户解释和处理。" |
| 禁忌话术 | 不要说"我们质量绝对没问题"这种绝对化表述；不要含糊说"有质检流程"却不展示具体报告模板 |
| 话术意图（四维度） | **情绪**：从"质量不可控的担忧"转为"出问题也能说得清楚"的安心感；**信息目标**：了解客户过去是否遇到过质量追溯困难的实际情况；**关系**：建立透明、可核实的专业形象；**行动导向**：引导客户提出想看具体质检报告样本，或主动进入小批量试单流程 |
| 升级路径 | 若客户对质检细节要求较高（如大批量采购前）：可安排产线参观或提供更详细的质检体系说明；若客户此时更关注价格而非质量细节：简要说明后转入报价环节，可参照S10阶梯报价与让步节奏表 |
| 使用注意 | 质检报告模板应提前打印或电子化准备好，现场能直接展示，不要只是口头描述 |

## 四、本模块专属验收细则

| 检查点 | 标准 |
|---|---|
| BEDS四要素完整 | 每个卖点是否完整包含B-E-D-S四段，不能跳过某一环节（尤其容易被跳过的是B背景） |
| D证明可验证 | 证明部分是否给出具体、可核实的事实依据，而非笼统说法 |
| S利益翻译到位 | 是否把优势翻译成客户能直接感知的实际好处，而非停留在"我们做得好"层面 |
| 卖点数量 | 至少覆盖2个核心卖点，避免只讲一个显得单薄；如企业有更多核心卖点，应按BEDS结构自行补充 |
| 卖点选择依据是否掌握 | 现场演练时检查学员能否根据客户初期信号判断优先讲哪个卖点，而非机械地把所有卖点一次性讲完 |

---

## 五、课件页要点

1. 讲清楚BEDS四要素的逻辑关系：背景建立共鸣→优势给出做法→证明取得信任→利益让客户明白"关我什么事"，四步缺一不可。
2. 讲解"卖点选择依据"：不是把所有卖点倒出来，而是先听、再判断该讲哪个。
3. 现场练习：让学员选择本企业一个真实的产品卖点，按BEDS结构现场组织话术，导师用验收细则点评是否四要素完整、证明是否可验证。
